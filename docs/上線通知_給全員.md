## 🛠️ 步驟一：建基礎設施 — 只有 R4 做（吳瑾瑜，備援 郝家銘，約 30–60 分）

照 SOP **從 §3.1 開始**依序跑：

1. §3.1 設定 project + 開 API（含 Cloud Run、Secret Manager）
2. §3.2 建 raw bucket `gs://polaris-desk-raw`
3. §3.3 建 canonical dataset `polaris_core`
4. §3.4 收齊 7 人 gmail 後，逐一綁 `READER` + `bigquery.jobUser`
5. §3.5 設預算告警 + per-user 查詢配額（**billing account ID 跟 PM 拿，別貼進 public repo**）
6. §4 建分區表 + 向量索引 + 跑一次 ingestion
   - ⚠️ 先確認 `asia-east1` 支援 `CREATE VECTOR INDEX`；注意「**少於 5,000 列不建索引**」的限制
7. 完成後在群組喊一聲：**「canonical 就緒」**

---

## 🚀 步驟二：每人上線 onboarding（全員，等 R4 喊就緒後，約 10 分／人）

照 SOP **§5** 做四步：

1. `gcloud auth login` + `gcloud auth application-default login` + 設 project
2. 建自己的 scratch：`polaris_dev_<你的英文名>`（例：`polaris_dev_wayne`）
3. 在本機 `.env` 設 `DEV_DATASET`（**別 commit `.env`**）
4. 跑驗證查詢，能回傳數字 = 成功 🎉

---

## 📏 日常三守則（全員，請記牢）

1. **讀 `polaris_core`、寫 `polaris_dev_<name>`** — 永遠不要寫 core。
2. 每次查詢帶 `published_at` 範圍 + `stock_id`；大查詢前先 `--dry_run` 估成本。
3. **不要**自己重建 index / 重跑 ingestion。要改資料或加欄位 → 走 §7 的 PR 流程（R2 審、PM 核、R4 套用）。
