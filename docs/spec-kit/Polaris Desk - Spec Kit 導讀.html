01 · The Ruler

我們用什麼尺在量

Spec Kit 的核心精神：需求要「可驗收」。每條任務都要過下面 6 關，否則就是寫得不夠清楚。

可量測完成 = 數字 / 可勾的事實，不是「polish / 穩定 / review」
可驗收用 Given / When / Then 或一個門檻說「怎樣算過」
可追溯對應到某個 FR / NFR / Go-No-Go 閘
一致A 說靠 B，B 也真的有交給 A，時間對得上
無缺口沒有「沒人認領」的工作
標出未決沒拍板的標 [NEEDS CLARIFICATION] + 到期 + owner
02 · Constitution

專題憲法 — 不可違反

凌駕所有任務。任何 user story / PR 違反即 No-Go。（repo 內權威版：.specify/memory/constitution.md）

I.合規紅線 NFR-031NON-NEGOTIABLE

新聞 / 投研功能只描述、標證據、標矛盾，不得產出任何買賣建議（投顧執照風險）。目標：130 題 + 4 場景 + 紅隊題中買賣建議 = 0。

II.引用接地 GroundingNON-NEGOTIABLE

每句結論 / 每個數字都要可追溯來源（頁碼 / 出處 / 財報欄位）。無源不輸出；來源矛盾就標出矛盾、不替使用者下結論。

III.本地優先 · 金鑰安全

開發先本地 pgvector 省雲費（Q-03）；金鑰只放 .env / Secret Manager，永不 commit、不貼群組、不丟 Drive（Q-10）。

IV.Eval 即品質門檻

好不好以 Ragas + 三方 Judge 客觀分數為準。CP ≥ 0.85、Faithfulness ≥ 0.90、AR ≥ 0.85；130 題達標 ≥ 80%（G3）。三方 Judge 只在閘門跑、CI 用便宜模型（省 token）。

V.Demo 可重現 + 離線備援

Demo 跑雲端，但必有「本地 pgvector + 預錄影片」Plan B；同場景跑 10 次一致；斷網 < 30 秒切備援。

VI.最新技術棧

google-genai 新 SDK + gemini-3-pro/flash-preview + gemini-embedding-2（768/cosine）；非舊版 google-generativeai。Rerank 用 Cohere rerank-v4.0。

03 · User Scenarios

4 個 Demo 場景 = 4 個 User Story

具體問句由 R1 在 W2 Day 6–8 細寫（草稿見 _demo場景草稿_給R1.md）。每個場景都受 NFR-031 約束。

US1 — 單一公司投研摘要

P1
Given 已入庫某公司某季法說稿，When 問「該季營運重點」，Then 回 ≥ 3 點摘要、每點帶可點擊跳轉來源、且無買賣建議。問未入庫期間則答「資料未涵蓋」、不編造。

US2 — 同業比較（Deep Research Agent 主秀）

P1
Given A、B 兩公司已入庫，When 問「比較毛利率」，Then 並列數字 + 各自來源 + 一句中性差異描述。Deep Research ≤ 6 ReAct 迴圈、≥ 3 引用、可溯源。

US3 — 多模態圖表問答（ColPali）

P2 · 可砍
Given 含圖表的法說 PDF，When 問圖表數據，Then 正確回答 + 指出「第 N 頁圖表」。門檻：圖表題命中率 ≥ 70%（否則砍，記 TD-01）。G3 ColPali 失敗則連本場景砍。

US4 — 跨產業營收拆解

P1
Given 多產業公司已標 schema，When 問「各事業群營收占比」，Then 回分產業拆解 + 來源。附加產業門檻依 AQ-08（建議 10–15%）。有獨立 10 題 eval（AQ-06）。

跨場景亮點 — Watchdog 事件驅動合規

真 Agent #2
Given 一則模擬 MOPS 公告，When 事件觸發，Then Watchdog 在 Alert Inbox 產出合規判斷摘要，且不出買賣建議。
04 · Success Criteria

技術無關、可被驗證的數字。這些就是「做完了沒」的客觀答案。

SC-006 LLMLingua 量到 token 省 ≥ 50%（目標 ~60%）
SC-007 結論句引用覆蓋率 = 100%
