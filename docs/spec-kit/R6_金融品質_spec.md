# 角色規格書（Spec Kit）：R6 — 金融品質工程師

**Role**: R6 台股 Ontology + 金融事實校對 + NFR-031 紅隊 **色**：紫
**對應**：專題 spec FR-005/006、Constitution I/II；4 週計畫 R6 卡 **Status**: Draft

## 1. Mission
建台股「分類字典」（Ontology）並守正確性與合規：W1 全隊最吃重，產出 Ontology v1；之後每日校對金融事實、出紅隊題攻 NFR-031、標註場景 4。
- **In scope**：Ontology（公司 / 財務指標 / 事件 / 法遵名詞）、多產業分類、金融事實校對、紅隊題、新聞白名單 / 可信度分級、場景 4 標註。
- **Out of scope**：寫程式（提供規則 / 內容給 R3/R4/R5）、評分工具（R5）。

## 2. Deliverables & Acceptance Criteria

| 交付物 | 對應 | 可驗收標準（measurable）|
|---|---|---|
| Ontology v1（凍結）| G1 | 含首批 20 家公司、四大類字典皆有實例可查、含多產業分類 schema（FR-074~076）|
| 紅隊題庫 | SC-003/G2 | 設計誘導買賣建議 / 誘導幻覺的題；跑一輪攻擊系統 |

| 金融正確性把關 | SC-003 | **抽查 ≥ 30 題財務數字錯誤率 = 0**；**130 題 + 4 場景買賣建議 = 0** |

## 4. Dependencies
- **上游**：R1（一起查公司）。
- **下游**：R4 把你的 Ontology 灌進庫；R5 用你的題 / 標註；R3 的新聞卡 / 合規規則靠你定。

## 5. Risks & Fallback
- Ontology 是 G1 地基且最吃重 → W1 全力投入、Day 4 出 v1 候選；沒凍結則 G1 No-Go（延 1 天）。
- AQ-08 多產業門檻未定 → Day 4 與 R2 拍板（建議 10–15%）。

## 6. Constitution 遵循
- **I（NFR-031）紅隊總攻手**：專攻「能不能誘導系統說買 / 賣」，目標讓違規數 = 0。
- **II**：校對時要求系統每個金融數字都有來源，無源即記為錯誤。
