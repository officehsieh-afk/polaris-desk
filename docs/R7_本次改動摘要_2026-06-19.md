## Issue #118 — /ask adapter 對齊後端契約 + NFR-031 surface

### 修改檔案

**`frontend/src/types/api.ts`**
- 新增 `AskCitationRaw`、`NodeTraceRaw` interface（對齊 live 後端欄位）
- 改寫 `AskResponse`：`{ answer, compliance_status, citations: AskCitationRaw[], trace: NodeTraceRaw[] }`
- 新增 `ChunkRaw` interface（對應 GET /chunk/{source_id} 契約）

**`frontend/src/types/viewmodel.ts`**
- `AskVM` 加入 `compliance_status: string` 欄位

**`frontend/src/lib/adapters.ts`**
- `normalizeAsk(raw, query)` 重寫：從 `raw.answer` 解析 bullets、從 `raw.citations` 建 summary、`raw.trace` 轉 ReAct steps
- `normalizeResearch` 補上 `compliance_status: raw.compliance_status`
- 新增 helper：`nodeTracesToReact()`、`askCitationToTracker()`

**`frontend/src/app/(dashboard)/research/page.tsx`**
- `handleOpenDoc` 改為 async：優先呼叫 `api.chunk()` 取後端真實文件，fallback 本地 DOC_CONTENTS，再 fallback VM snippet
- `compliance_status === "blocked"` 條件渲染：blocked 時只顯示 `<ComplianceBanner message={後端安全訊息}>`、抑制 kpis/summary/chart 輸出（NFR-031）
- 修正因插入 blocked ternary 造成的 JSX fragment 未關閉語法錯誤
- `data?.chart` 全面改為 `displayData?.chart`

## Issue #114 — Google 登入 + /history B 級 + Magic Link 移除

### 新增檔案

**`frontend/src/app/api/auth/[...nextauth]/route.ts`**（新建）
- NextAuth v4 + Google Provider
- JWT callback：將 `account.id_token` 存入 token
- Session callback：將 `idToken` 暴露到 session

**`frontend/next.config.ts`**（新建）
- `output: "standalone"` — Cloud Run 部署必需

**`frontend/Dockerfile`**（新建）
- 三階段 build（deps / builder / runner）
- ARG `NEXT_PUBLIC_API_BASE` 建置時注入
- Cloud Run 以 `$PORT` 注入監聽埠（預設 3000）

### 修改檔案

**`frontend/src/app/providers.tsx`**
- 加入 `SessionProvider`（next-auth/react）包住 `ThemeProvider`

**`frontend/src/lib/api.ts`**
- 新增 `authHeaders()`：呼叫 `getSession()`，有 idToken 則回 `{ Authorization: Bearer <token> }`，否則回 `{}`
- `realFetch()` 加入 `authHeaders()` spread
- `api.history()`：登入時呼叫 `GET /history`，未登入 fallback localStorage
- 新增 `api.historyOne(id)`：`GET /history/{id}`
- 新增 `api.postHistory()`：fire-and-forget，401 靜默忽略
- 新增 `api.chunk(sourceId)`：`GET /chunk/{sourceId}`，回傳 `DocContent | null`

**`frontend/src/app/(dashboard)/settings/page.tsx`**
- 移除 Magic Link 表單（email/sent state、divider、form）
- Google 按鈕接 `signIn("google")` / `signOut()`
- 顯示真實使用者名稱、email、頭像（來自 `session?.user`）

**`frontend/src/components/layout/AppShell.tsx`**
- `useSession` 取得真實使用者
- AppShell 左下角：顯示真實 `name`、`image`，計算 initials；無 session 顯示「訪客」

**`frontend/src/app/(dashboard)/history/page.tsx`**
- `handleItemClick`：登入時呼叫 `api.historyOne(id)`，結果存 `sessionStorage("polaris_restore")`，導到 `/{page}?historyId=`；未登入 fallback `?q=` 重跑
- 列表項改為 `<button onClick={handleItemClick}>`

**`frontend/src/app/(dashboard)/research/page.tsx`**（同上，另含）
- `restoredData` state + `useEffect`：頁面載入時讀 `sessionStorage("polaris_restore")` 還原歷史結果
- `displayData = restoredData ?? data`

**`frontend/src/app/(dashboard)/peer/page.tsx`**
- 結果取得後呼叫 `api.postHistory("peer", ...)` 寫入後端歷史
