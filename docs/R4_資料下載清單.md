# R4 資料下載清單 — 要灌進 Polaris Desk 的語料

> **目的**：一張清單講清楚「要下載哪些資料、給 agent 解析(parsing)、索引(indexing)進向量庫/資料表」。
> **權威來源**：[`R4_ingestion_開工指南.md`](R4_ingestion_開工指南.md)（pipeline 細節）、[`../specs/001-langgraph-skeleton/data-model.md`](../specs/001-langgraph-skeleton/data-model.md)（欄位）、`.specify/memory/constitution.md`（憲法接地原則）。
> **排序原則**：依「解鎖全隊的關鍵路徑」由先到後（W1 法說 → 財報 → MOPS → ColPali → 新聞）。

## 硬約束（下載 / 入庫前先讀）

- 🔴 **NFR-031**：任何語料的描述 / 分析**只能描述事實、標證據、標矛盾，不得產出買賣建議**。
- **引用接地**：每筆 chunk / 每個數字都要帶 `source_id` 可溯源（FR-003）。
- **embedding 只算一次、全隊共讀**（`gemini-embedding-2`，768 維 / cosine）；別重算。
- **先寫自己的 `polaris_dev_<name>` 驗過，再進 `polaris_core`**（canonical 全隊唯讀依賴）。
- **金鑰走 ADC / `.env`，永不 commit。**

## 下載的檔案放哪裡（重要：**不放 git repo**）

原始 PDF 的單一真實來源是 **GCS bucket `gs://polaris-desk-raw`**（SOP §4「檔案層單一真實來源」）。repo 的 `.gitignore` 已擋 `*.pdf` 與 `data/`，**PDF 不可 commit 進 repo**（避免灌爆 git、且全隊共讀同一份雲端檔）。

| 層 | 位置 | 放什麼 | 誰寫 |
|---|---|---|---|
| **原始檔（雲端）** | `gs://polaris-desk-raw` | 下載的 PDF 本體 | 全隊上傳；R4 ingestion 讀 |
| **canonical（雲端）** | BigQuery `polaris_core` | 解析後 `chunks` / `financial_metrics` / `colpali_index` | **只有 R4 可寫** |
| **本機暫存** | `./data/`（已 gitignore） | 跑 ingestion 的本機工作副本 | 個人，不進 git |

**bucket 內資料夾慣例**（沿用既有命名，metadata 推導靠它）：

```
gs://polaris-desk-raw/
├─ 07_ConferenceCall/2330_TSMC/<...Transcript....pdf>   ← 軌 1 法說逐字稿 / 簡報
├─ 06_Financial_Report/2330_202503.pdf                  ← 軌 2 財報（檔名 YYYYMM）
├─ 08_MOPS/        ← 軌 3（新增）月營收 / 重訊
└─ 09_News/        ← 軌 5（新增）新聞，檔名或 manifest 帶 URL
```

**流程**：下載 PDF → `gsutil cp` 上傳到對應軌資料夾 → R4 ingestion 從 bucket 讀 → 解析/切塊/embedding → 先寫 `polaris_dev_<name>` 驗證 → 過了再進 `polaris_core`。本機只留 `./data/` 暫存、不 commit。

## 軌 2｜財報（W1 後段–W2，**兩條 pipeline**）

財報價值在**精確數字**，不能靠向量檢索硬抓 → 分兩軌。**逐頁先偵測有無文字層再分流**（`pdftotext -f p -l p` 抽到 ~0 字 = 圖檔頁，走 vision）。

### 軌 2A｜結構化數字 → `polaris_core.financial_metrics`（餵 Calculator）

| 要下載什麼 | 來源 | 注意 |
|---|---|---|
| 季報/年報**財務報表**（損益表、資產負債表、現金流量表）| MOPS 財務報表 / 公司網站 | ⚠️ **2330 報表頁 400dpi 掃描圖、0 文字層 → Gemini vision OCR**；2454/3034 有文字層可直抽 |

- 要抽的指標：營收、營業成本、營業毛利、毛利率、營業費用、營業利益、稅前/稅後淨利、EPS、每股淨值、總資產/負債/權益。

## 每筆入庫必帶 metadata（接地剛性要求）

| 欄位 | 說明 |
|---|---|
| `stock_id` | 2330 / 2317 …（從資料夾/檔名推） |
| `fiscal_period` | **統一格式 `2024Q3`**，對齊 [`../src/polaris/graph/temporal.py`](../src/polaris/graph/temporal.py)；法說 `1Q24→2024Q1`、財報 `202503→2025Q1` |
| `doc_type` | `transcript` / `presentation` / `financial_report` / `news` |
| `source_id` | FR-003 逐句溯源 |
| `published_at` | **partition key，務必填**；無日期用季末日（Q1→03-31…） |
| `embedding` | `gemini-embedding-2`，768 維 / cosine，只算一次 |
