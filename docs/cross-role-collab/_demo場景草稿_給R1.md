## 場景 1 — 單一公司投研摘要（P1 · 信任的起點）

- **示範問句**：「`[填: 公司, 如 台積電]` 在 `[填: 2024Q3]` 法說會講了哪些營運重點？」
- **期望輸出**：3–5 點摘要，**每點帶可點擊跳轉的來源**（法說稿第 N 頁），結尾標「以上為事實摘要，非投資建議」。
- **要秀的亮點**：引用接地 + Citation Tracer 點擊跳原文。
- **對應**：FR-001/003、SC-007。
- **NFR-031 自查**：不得出現「建議買進 / 看好 / 目標價」等字眼。
- **變體（展示誠實）**：故意問未入庫季別 → 系統答「資料未涵蓋」而非編造。

## 場景 2 — 同業比較（P1 · Deep Research Agent 主秀）

- **示範問句**：「比較 `[填: A 公司]` 與 `[填: B 公司]` 最近 `[填: 兩季]` 的毛利率變化。」
- **期望輸出**：並列數字表 + 各自來源 + 一句**中性**差異描述；由 Deep Research Agent 跑 ReAct（想→查→想）。
- **要秀的亮點**：ReAct trace UI（agent 思考動線）+ Calc grounding（數字有來源）。
- **對應**：US2、FR-004/005、SC-001。
- **驗收門檻**：Deep Research ≤ 6 迴圈、≥ 3 引用、結論可溯源。
- **NFR-031 自查**：只描述差異，不得說「A 比 B 值得買」。

## 場景 3 — 多模態圖表問答（P2 · 可砍，ColPali 亮點）

- **示範問句**：「`[填: 公司]` 法說 PDF 第 `[填: N]` 頁那張 `[填: 營收結構圖]` 顯示什麼？」
- **期望輸出**：正確讀出圖表數據 + 指出「出自第 N 頁圖表」。
- **要秀的亮點**：ColPali 多模態檢索（用「看」的理解圖表）。
- **對應**：US3、TD-01。
- **前提 / 降級**：ColPali POC 圖表題命中率 ≥ 70% 才採用；**G3 若 ColPali 失敗 → 砍本場景 + ColPali**。

## 場景 4 — 跨產業營收拆解（P1 · 有獨立 10 題 eval）

- **示範問句**：「拆解 `[填: 鴻海]` 各事業群 / 產業的營收占比，並指出主要與附加產業。」
- **期望輸出**：分產業拆解表 + 來源；附加產業占比門檻依 **AQ-08（建議 10–15%）**。
- **要秀的亮點**：多產業 schema（`gics_classifications`）+ 跨文件對齊。
- **對應**：US4、FR-074~076、AQ-06（場景 4 獨立 10 題 eval）、AQ-08。
- **NFR-031 自查**：只呈現營收結構，不得對任何事業群下「看好 / 該買」結論。

## 跨場景亮點 — Watchdog 事件驅動合規（真 Agent #2）

- **演法**：丟一則**模擬 MOPS 公告** → Watchdog 自動觸發 → Alert Inbox 跳出合規判斷摘要。
- **要秀的亮點**：cron → event-driven 的升級（部落格 3 主題）。
- **NFR-031 自查**：合規摘要只標「有無合規疑慮 + 證據」，不得出買賣建議。
