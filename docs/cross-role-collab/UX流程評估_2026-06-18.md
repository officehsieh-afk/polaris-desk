## 預期使用者旅程

```
登入
 └─ 研究助理：輸入問題 → 看有來源的分析 → 查看引用原文
     ├─ 監控警示面板：自動偵測異常
     ├─ 送到同業比較：比較兩家公司的財務與法說
     └─ 訂閱公司 → 通知中心追蹤動態
         └─ 查看對話紀錄 / 研究資料庫
```

## 🔴 邏輯有斷點（使用者會感知到功能壞掉）

### 1. 假登入狀態貫穿整個 app

AppShell 顯示 hardcoded「Jing Chen」，Google OAuth 按鈕點了沒反應，Magic Link 表單送出後只在前端 mock。使用者以為自己登入，但沒有身份識別，後續所有 user-level 功能（history 跨裝置、subscriptions）也因此無法正常運作。

**影響範圍**：所有頁面（AppShell 常駐）

---

### 2. 引用接地是核心承諾，但目前是降級模式

憲法要求「每個數字都要有來源」，這是整個產品的信任基礎。`GET /chunk/{source_id}` 尚未實作，DocViewer 顯示的是 snippet 切句、hardcoded trust score「mid」，不是真實原文與頁碼。

| | 現在 | `GET /chunk/{source_id}` 交付後 |
|--|------|------|
| 引用內文 | snippet 切句（1–3 行） | 完整段落原文 |
| 頁碼 | 來自 `published_at` 或 snippet | 真實頁碼 |
| 可信度標籤 | hardcoded `"mid"` | 後端計算的 trust score |

**影響範圍**：研究助理頁、同業比較頁

---

### 3. 量化分析區塊永遠是佔位元件

研究助理頁的 KPI 卡和長條圖永遠顯示「財務指標資料建置中」。但 BQ 的 `financial_metrics` 表已有 4,368 筆資料，資料在，只差後端 workflow 附在 `/research` response 裡。使用者的財務分析需求在這個頁面沒有得到回應。

**影響範圍**：研究助理頁

---

### 4. 同業比較整頁是 mock，外觀與真實資料一模一樣

`POST /peer-compare` 尚未實作，整頁顯示 hardcoded 台積電 vs 聯發科數字。使用者輸入任何兩家公司組合，畫面仍顯示同一組假資料，沒有任何「測試資料」提示。在財務工具中，假數字被誤讀為真實分析是信任風險，也可能觸及 NFR-031 合規邊界。

**影響範圍**：同業比較頁（整頁）

---

### 5. 監控警示面板完全空白，無法區分「沒異常」與「功能未完成」

- `GET /alerts` 缺 `origin` 欄位 → 兩個頁面的面板顯示 0 筆
- `POST /contradiction` 不存在 → 矛盾偵測走 client-side 規則式 mock

空白面板沒有任何說明文字，使用者無法判斷是系統正常（沒有異常）還是功能未完成。

**影響範圍**：研究助理頁、同業比較頁

## 補齊優先建議（供開會討論）

> 非指派，供排工時時參考。分為「已追蹤待交付」與「遺漏需新增追蹤」兩類。

### 已追蹤、調整優先順序

| 優先 | 端點 / 功能 | 原因 |
|------|------------|------|
| 🔴 1 | `GET /chunk/{source_id}` | Demo 場景 1 主秀亮點；引用跳原文是信任基礎，目前降級 |
| 🔴 2 | `POST /peer-compare` | Demo 場景 2 P1 主秀；整頁 mock 是最大 demo 風險 |
| 🔴 3 | `GET /alerts` 補 `origin` | 三行改動即可修復；兩頁面板空白比沒功能更令人困惑 |
| 🔴 4 | `POST /contradiction` | 矛盾偵測是研究流程安全網，目前 client mock 有誤導風險 |
| 🟡 5 | `POST /research` 補 `kpis` / `chart` | 資料在 BQ，只差 workflow 附上；量化分析永遠空白讓核心價值打折 |
| 🟡 6 | `GET /library` | 使用者不知道能問什麼，降低摩擦 |
| ⚪ 7 | Auth 真實串接 | 影響跨裝置，但不影響核心分析流程 |

### 遺漏、需新增追蹤

| 優先 | 項目 | 建議動作 |
|------|------|---------|
| 🔴 Demo 前必確認 | 同業比較頁加「示範資料」標示 | R7 可立即加，一行 UI 改動，避免 NFR-031 風險 |
| 🔴 Demo 前必確認 | NFR-031 disclaimer 是否每次都帶 | 確認 `POST /research` response 或前端固定顯示，並寫入驗收條件 |
| 🔴 Demo 前必確認 | Demo Watchdog 觸發機制 | 確認如何在 demo 現場模擬 MOPS 公告觸發警報 |
| 🟡 需確認 | `gics_classifications` 表是否存在 | 場景 4 依賴它；若不在 BQ 需補建，`company_dim.industry_id` 是否足夠替代需評估 |
| 🟡 需確認 | ColPali HTTP 端點 | 5,701 筆資料在 BQ 無人取用；場景 3 備用但需端點；由誰實作需確認 |
| 🟡 需確認 | 邊緣誠實（未入庫 ticker / 季次） | 查詢不在 BQ 的資料，系統應回「資料未涵蓋」而非亂編，目前行為未測試 |
